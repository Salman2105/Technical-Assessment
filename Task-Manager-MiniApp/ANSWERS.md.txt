# 2. Stack choice

I chose the MERN stack (MongoDB, Express, React, Node.js) because it allows fast full-stack development using JavaScript across both frontend and backend.

React was useful for building reusable UI components and handling dynamic state updates for task creation, editing, filtering, and search.

Express and MongoDB provided a simple and flexible backend structure for CRUD operations.

Tailwind CSS helped speed up responsive UI development while keeping styling organized.

A worse choice for this task would have been a heavier enterprise framework such as Java Spring Boot because it would increase setup complexity and development time for a relatively small CRUD-focused application.

# 3. One real edge case

The application correctly handles overdue tasks.

File:

```txt
frontend/src/components/TaskCard.jsx
```

Logic:

```js
const isOverdue =
  new Date(task.dueDate) < new Date() &&
  task.status !== "completed";
```

This ensures completed tasks are not incorrectly marked as overdue even if the due date has passed.

Without this handling, completed tasks would still display an overdue badge, creating incorrect task status behavior and confusing the user.
